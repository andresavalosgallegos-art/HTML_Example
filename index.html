Cómo convertir un archivo .py a .exe y crear un instalador

Guía paso a paso y detallada para compilar tu aplicación Python (con GUI) en un ejecutable de Windows usando **PyInstaller** y generar un instalador con **Inno Setup**.

Paso 1: Código de ejemplo (Calculadora simple)

Usaremos el siguiente código Python, guárdalo como calculadora.py:

import tkinter as tk
from tkinter import messagebox

def click_boton(valor):
    actual = str(entrada.get())
    entrada.delete(0, tk.END)
    entrada.insert(0, actual + valor)

def limpiar():
    entrada.delete(0, tk.END)

def calcular():
    try:
        resultado = eval(entrada.get()) 
        entrada.delete(0, tk.END)
        entrada.insert(0, str(resultado))
    except:
        messagebox.showerror("Error", "Operación inválida")

ventana = tk.Tk()
ventana.title("Calculadora GUI")
ventana.geometry("300x400")

entrada = tk.Entry(ventana, width=16, font=("Arial", 24), borderwidth=2, relief="solid", justify="right")
entrada.grid(row=0, column=0, columnspan=4, padx=10, pady=10)

botones = [
    "7", "8", "9", "/",
    "4", "5", "6", "*",
    "1", "2", "3", "-",
    "0", ".", "=", "+"
]

fila = 1
col = 0

for boton in botones:
    if boton == "=":
        tk.Button(ventana, text=boton, width=5, height=2, font=("Arial", 18),
                  command=calcular).grid(row=fila, column=col, padx=5, pady=5)
    else:
        tk.Button(ventana, text=boton, width=5, height=2, font=("Arial", 18),
                  command=lambda b=boton: click_boton(b)).grid(row=fila, column=col, padx=5, pady=5)
    col += 1
    if col > 3:
        col = 0
        fila += 1

tk.Button(ventana, text="C", width=22, height=2, font=("Arial", 18), command=limpiar).grid(row=5, column=0, columnspan=4, padx=5, pady=5)

ventana.mainloop()
Descargar calculadora.py

Paso 2: Instalar y Verificar PyInstaller

Abre tu terminal o CMD y ejecuta la instalación:

pip install pyinstaller

Verificación:

Asegúrate de que la instalación fue exitosa ejecutando el comando de versión. Si ves un número de versión, estás listo:

pyinstaller --version

Paso 3: Personalización Avanzada y Conversión (.py a .exe)

Para aplicaciones de producción, rara vez usarás solo el comando básico. A continuación, se detallan las opciones más importantes que puedes agregar al comando pyinstaller:

Opciones Fundamentales para el EXE

Este es el comando que usaremos para compilar la calculadora:

pyinstaller --onefile --noconsole --icon=calculadora.ico calculadora.py
  • --onefile: Crea un **único archivo ejecutable (.exe)**. Esto es ideal para la distribución, ya que el usuario solo ve un archivo. (El inconveniente es que tarda un poco más en arrancar).
  • --noconsole o -w: **Oculta la ventana de la consola (CMD)**. Es esencial para aplicaciones con interfaz gráfica (GUI) como las hechas con Tkinter o PyQt, ya que no queremos que el usuario vea una ventana negra.
  • --icon=FICHERO.ico o -i FICHERO.ico: **Añade un icono al ejecutable**. Necesitas un archivo con la extensión **`.ico`**. Si no especificas la ruta, PyInstaller usará un icono genérico de Python.

Añadir Archivos de Datos (Imágenes, Bases de Datos, etc.)

Si tu script necesita acceder a archivos externos (como imágenes, archivos de configuración o una base de datos SQLite), debes decirle a PyInstaller que los incluya en el paquete. Para ello, se usa la bandera --add-data:

Sintaxis General:

--add-data "ruta_origen;nombre_carpeta_destino"

Ejemplo (Windows): Si tu código usara una imagen llamada logo.png:

pyinstaller --onefile calculadora.py --add-data "logo.png;."

Esto copia logo.png al directorio raíz del ejecutable empaquetado (el punto . indica el directorio raíz).

Cómo Acceder al archivo en tu código Python:

Cuando un script se ejecuta dentro de un EXE de PyInstaller, la ruta de los archivos cambia. Debes usar la biblioteca sys para obtener la ruta correcta en tiempo de ejecución. Debes agregar este código al inicio de tu script Python:

import sys
import os

def resource_path(relative_path):
    """Obtiene la ruta absoluta al recurso para PyInstaller."""
    try:
        # PyInstaller crea una carpeta temporal y guarda la ruta en _MEIPASS
        base_path = sys._MEIPASS
    except Exception:
        base_path = os.path.abspath(".")

    return os.path.join(base_path, relative_path)

# Ejemplo de uso:
# ruta_logo = resource_path('logo.png')

Resultado del Paso:

Ejecuta tu comando con las opciones deseadas (por ejemplo, el comando del principio de este paso). El ejecutable calculadora.exe se encontrará en la nueva carpeta **`dist`**.

Paso 4: (Opcional) Crear un instalador con Inno Setup

  1. Descarga e instala Inno Setup desde aquí.
  2. Crea un script básico como este (setup.iss) en la raíz del proyecto. Asegúrate de que el icono del instalador esté incluido si creaste uno:
; Script de ejemplo para Inno Setup
[Setup]
AppName=Calculadora Simple
AppVersion=1.0.0
DefaultDirName={autopf}\CalculadoraSimple
OutputDir=.\InstallerOutput
; Si tienes un icono, puedes ponerlo aquí
SetupIconFile=calculadora.ico 

[Files]
; Source apunta al EXE generado por PyInstaller
Source: "dist\calculadora.exe"; DestDir: "{app}"; Flags: ignoreversion

[Icons]
Name: "{autodesktop}\Calculadora Simple"; Filename: "{app}\calculadora.exe"
  1. Abre el script en Inno Setup y haz clic en “Compile”.
  2. ¡Listo! El instalador se generará en la carpeta `InstallerOutput`.

⚠️ Solución de Errores Comunes al Instalar PyInstaller

Si la instalación en el **Paso 2** falló, revisa los siguientes problemas comunes en detalle:

1. Error: pip no se reconoce (Problema de PATH)

Si recibes 'pip' is not recognized... (Windows) o pip: command not found (Linux/macOS), el problema es que el sistema operativo no puede localizar la herramienta pip o los comandos de Python/PyInstaller.

Solución Detallada:

  • Windows: Esto ocurre si no marcaste la casilla **"Add Python to PATH"** durante la instalación. Debes añadir manualmente las rutas de Python y la carpeta de Scripts a las variables de entorno o, de forma más simple, reinstalar Python marcando la opción.
  • Entornos con múltiples Pythons: Si tienes Python 2 y 3 instalados, el comando principal `pip` podría apuntar a la versión incorrecta. Usa el comando específico: 
    pip3 install pyinstaller

2. Error: Permisos Denegados (`Permission denied`)

Si recibes un error de **Permission denied** o **Could not install packages due to an EnvironmentError**, significa que estás intentando instalar en un directorio del sistema protegido (como `/usr/local/` o `C:\Program Files`).

Solución Detallada:

  • Opción recomendada (Instalación de Usuario): Instala PyInstaller solo para tu usuario. Este método no requiere permisos de administrador y es la mejor práctica: 
    pip install --user pyinstaller
  • Opción de Administrador: Ejecuta la terminal **como administrador** (en Windows) o usa `sudo` (en Linux/macOS) antes del comando: 
    sudo pip install pyinstaller

3. Error: Versión de Python Incompatible o Versión Antigua de `pip`

Si ves errores como **Could not find a version that satisfies the requirement pyinstaller** o problemas con la compilación de ruedas (`wheel`), tu `pip` está obsoleto o la versión de Python es muy antigua.

Solución Detallada:

  • **Actualizar `pip`:** Siempre es una buena práctica actualizar el gestor de paquetes antes de instalar: 
    python -m pip install --upgrade pip
  • **Versión de Python:** Asegúrate de usar una versión de Python moderna (3.8+). Si tu versión es muy antigua, PyInstaller podría no soportarla.

Resumen

  1. Escribe el script Python (calculadora.py).
  2. **Personaliza** el ejecutable: **`pyinstaller --onefile --noconsole --icon=calculadora.ico calculadora.py`**.
  3. Usa Inno Setup para crear el instalador.
